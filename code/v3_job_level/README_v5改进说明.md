# v5版本改进说明（基于专业反馈）

## 核心问题

v4版本存在的问题（专业反馈指出）：
1. **重复计分**: rmse和weighted_rmse高度相关
2. **低基数爆炸**: 相对误差在小值时不稳定
3. **尺度不一致**: 各项损失的权重难以直觉控制
4. **假性分段**: 只有线性权重，没有真正的断点建模
5. **过度裁剪**: 增长率限制30%太严格，可能掩盖真实动态

## v5核心改进

### 1. 使用SMAPE替代相对误差
```python
# v4: 低基数时爆炸
relative_error = (pred - actual) / actual

# v5: 对称稳定
smape = 2 * |pred - actual| / (|pred| + |actual| + epsilon)
```

### 2. 合并重复计分项
```python
# v4: 重复惩罚
total_loss = rmse * 0.4 + weighted_rmse * 0.3 + ...

# v5: 简化
total_loss = smape * 0.3 + weighted_smape * 0.3 + ...
```

### 3. 简化权重结构
```python
# v4: 嵌套放大（trend_loss * 10 * 0.3）
trend_loss = abs(pred_trend - data_trend) * 10
total_loss = ... + trend_loss * 0.3

# v5: 直接放大
trend_loss = abs(pred_trend - data_trend)
total_loss = ... + trend_loss * 2.0
```

### 4. 改进方向一致性计算
```python
# v4: 零变化会污染结果
direction_agreement = sign(actual_diff) * sign(pred_diff)

# v5: 只计算显著变化点
threshold = std(actual_diff) * 0.1
significant_mask = abs(actual_diff) > threshold
direction_agreement = sign(actual_diff[mask]) * sign(pred_diff[mask])
```

### 5. 放宽增长率限制
```python
# v4: 太严格（30%）
max_growth_rate = 0.3

# v5: 更合理（50%）
max_growth_rate = 0.5
```

### 6. 添加滚动回测功能
```python
# 验证预测稳定性
backtest_results = optimizer.rolling_backtest(
    window_size=12,  # 12季度训练窗口
    step=4           # 4季度滚动步长
)
```

## 诊断工具（最重要）

新增 `diagnose_model.py` 实现4项核心检查：

### 检查1: 滚动回测
- 验证不同训练窗口的预测稳定性
- 如果误差波动大 → 预测不可靠

### 检查2: 残差分析
- 检查系统性偏差（均值是否显著偏离0）
- 检查残差趋势（是否存在漂移）

### 检查3: 参数稳定性
- 不同窗口参数是否大幅跳变
- 跳变大 → 模型不可识别或过拟合

### 检查4: Baseline对比
- 与"上期值+平滑趋势"对比
- 如果不如baseline → 结构有问题

## 使用方法

### 第一步：诊断（必须先做）
```bash
cd v3_job_level
python diagnose_model.py
```

这会生成诊断报告，告诉你：
- ✓ 优化层改进可能有效
- ⚠️ 结构层问题（需要改模型）

### 第二步：根据诊断结果决定

**如果诊断显示"模型表现良好"**：
- 可以使用v5优化器改进拟合
- 运行完整预测

**如果诊断显示"结构层问题"**：
- 不要急于运行完整预测
- 根据具体问题类型采取对应措施

## 问题类型与解决方案

### 问题A: 系统性偏差/漂移
**症状**: 残差均值显著偏离0，或残差有明显趋势

**可能原因**:
- 数据口径变化（统计方法调整）
- 结构性断点（疫情、政策变化）
- 模型结构不匹配

**解决方案**:
1. 检查数据是否有口径变化
2. 考虑分段建模（2020-2021 vs 2022-2024）
3. 尝试其他模型结构

### 问题B: 参数不稳定
**症状**: 不同训练窗口参数大幅跳变

**可能原因**:
- 模型过于复杂，参数不可识别
- 数据不足以约束3个参数

**解决方案**:
1. 固定某些参数（如k=5）
2. 增加参数先验约束
3. 简化模型

### 问题C: 不如Baseline
**症状**: 复杂模型预测不如"上期值+趋势"

**可能原因**:
- 当前模型结构不适合该职位
- AI影响在该职位不显著

**解决方案**:
1. 对该职位使用简单时间序列模型
2. 检查该职位是否真的受AI影响
## 与v3/v4的对比

| 特性 | v3 | v4 | v5 |
|------|----|----|-----|
| 目标函数 | 单一RMSE | 6项加权 | 4项简化 |
| 低基数处理 | ❌ 会爆炸 | ❌ 会爆炸 | ✓ SMAPE稳定 |
| 重复计分 | ❌ 无 | ❌ 有 | ✓ 已消除 |
| 权重可控性 | ✓ 简单 | ❌ 嵌套难控 | ✓ 直观 |
| 滚动回测 | ❌ 无 | ❌ 无 | ✓ 有 |
| 诊断工具 | ❌ 无 | ❌ 无 | ✓ 完整 |

## 关键建议

### 1. 必须先诊断
不要直接运行完整预测，先用 diagnose_model.py 检查1-2个典型职位。

### 2. 接受现实
如果诊断显示"结构层问题"，优化器再好也没用，需要改模型结构或分段建模。

### 3. 关注业务解释
参数的经济含义比拟合误差更重要。如果参数不稳定或不合理，说明模型可能在过拟合噪音。

## 下一步行动

1. **运行诊断**: python diagnose_model.py
2. **查看报告**: 检查诊断图表和结论
3. **根据结果决定**:
   - ✓ 模型良好 → 使用v5优化器
   - ⚠️ 结构问题 → 分段建模或换模型
   - ❌ 不如baseline → 使用简单模型

## 核心理念

**优化器只能在"你给的模型族"里找最优。如果模型族本身表达不了真实机制，再好的优化器也只是"让错误看起来更顺眼"。**

因此，v5的核心价值不是"更好的拟合"，而是**诊断工具**——帮你判断问题是优化层还是结构层。
