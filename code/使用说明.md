# 职位级生态协同进化模型 - 使用说明

## 📋 系统概述

本系统实现了一个**职位级（Job-Level）**的生态协同进化模型，用于预测生成式AI影响下各具体职位的就业规模变化。

### 核心特点
- ✅ 每个职位独立建模（如"Specialty Trade Contractors"）
- ✅ 基于历史数据的参数优化
- ✅ 时间序列预测功能
- ✅ 支持286个职位的并行处理

## 🔧 每个职位的参数

### 基础参数（从数据计算）
- `N0`: 初始规模（95%分位数）
- `r0`: 基准增长率（历史平均）
- `K0`: 基础承载力（N0 × 1.2）

### 可学习参数（通过优化获得）
- `theta`: AI敏感度 ∈ [-1, 1]
  - 负值：易被替代
  - 正值：被AI增强
- `Ac`: 临界AI水平 ∈ [0, 1]
- `k`: 渗透速率 ∈ [1, 10]
- `G`: 技能向量（5维）
- `cost_reduce`: 成本削减比例
- `epsilon`: 价格弹性

## 🎯 核心公式

### 动力学方程
```
dN/dt = r₀ · N · [1 - N/K_eff(t)]
```

### 有效承载力
```
K_eff(t) = K₀ · f_t(A, G) · f_c(A)
```

其中：
- `f_t`: 技术因子 = 1 + θ · [σ(A) - 0.5] · C
- `f_c`: 经济因子 = ε·c·A + (g₀ + η·A)

## 📊 结果解读

### 参数含义

**theta（AI敏感度）**
- -1.0 ~ -0.6: 高度替代敏感（如数据录入员）
- -0.6 ~ -0.2: 中度替代敏感（如CAD绘图员）
- -0.2 ~ +0.2: 中性/轻微影响
- +0.2 ~ +0.6: 中度增强敏感（如数据科学家）
- +0.6 ~ +1.0: 高度增强敏感（如AI工程师）

**训练误差**
- < 0.05: 优秀拟合
- 0.05 ~ 0.10: 良好拟合
- > 0.10: 需要调整参数或模型

## 📝 注意事项

1. **数据质量**: 确保历史数据完整且准确
2. **计算时间**: 完整运行286个职位需要较长时间
3. **参数约束**: 优化器会自动确保参数在合理范围内
4. **过拟合风险**: 训练误差过低可能表示过拟合

---

## 🆘 常见问题

**Q: 优化时间太长？**
A: 减少 `maxiter` 或 `popsize`，或使用演示模式

**Q: 预测结果不合理？**
A: 检查参数范围，调整优化器的 `bounds`

**Q: 某些职位无法优化？**
A: 可能是数据不足，系统会自动跳过
