# 快速开始指南

## 当前情况

您的v3模型预测结果显示：
- 误差波动大（-44%到+82%）
- 某些职位趋势预测不准
- 个别职位严重偏离

## 我创建了什么

### v4版本（不推荐直接使用）
- 多目标优化函数
- 但存在重复计分、低基数爆炸等问题
- 可能"看起来更好但不一定更对"

### v5版本（推荐）
- 修复了v4的结构性问题
- 使用SMAPE替代相对误差
- 简化权重结构
- **最重要：添加了诊断工具**

## 立即行动（3步）

### 第1步：运行诊断（5-10分钟）

```bash
cd v3_job_level
python diagnose_model.py
```

这会：
- 对第1个职位进行完整诊断
- 生成诊断报告和图表
- 告诉你问题是"优化层"还是"结构层"

### 第2步：查看诊断结果

打开生成的 `diagnosis_*.png` 图表，查看：
- 拟合效果如何
- 残差是否有系统性偏差
- 参数是否稳定
- 是否优于简单baseline

### 第3步：根据诊断决定

**情况A：诊断显示"✓ 模型表现良好"**
→ 可以使用v5优化器改进所有职位

**情况B：诊断显示"⚠️ 系统性偏差/漂移"**
→ 需要分段建模或检查数据口径

**情况C：诊断显示"⚠️ 参数不稳定"**
→ 模型过于复杂，需要简化或固定某些参数

**情况D：诊断显示"❌ 不如baseline"**
→ 当前模型结构不适合，建议用简单时间序列模型

## 专业反馈的核心观点

> "v4更像是'让拟合看起来更好'的调参器，而不是解决根本性结构问题的方案。"

因此v5的重点是：
1. 修复v4的技术问题（SMAPE、去重复计分等）
2. **提供诊断工具**，帮你判断问题根源
3. 避免"过拟合噪音"
