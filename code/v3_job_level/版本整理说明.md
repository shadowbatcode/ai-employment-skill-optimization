# 代码版本整理说明

## 目录结构

```
v3_job_level/
├── 核心模型文件（通用）
│   ├── job_model.py              # 职位模型（所有版本共用）
│   ├── time_series_predictor.py  # 时间序列预测器（所有版本共用）
│   ├── data_processor.py         # 数据处理器（所有版本共用）
│   └── competition_matrix.py     # 竞争矩阵构建器（所有版本共用）
│
├── v3版本（原始版本）
│   ├── parameter_optimizer_v3.py # v3优化器
│   ├── main_v3.py                # v3主程序
│   └── 说明：单一RMSE目标函数，快速但精度有限
│
├── v4版本（不推荐使用）
│   ├── parameter_optimizer_v4.py # v4优化器
│   ├── main_v4.py                # v4主程序
│   └── 说明：多目标优化，但存在重复计分等问题
│
├── v5版本（推荐使用）
│   ├── parameter_optimizer_v5.py # v5优化器（修复v4问题）
│   ├── diagnose_model.py         # 诊断工具（最重要）
│   └── 说明：修复结构问题，添加诊断功能
│
├── 对比和可视化工具
│   ├── compare_v3_v4.py          # v3和v4对比工具
│   └── integrate_and_visualize.py # 数据整合和可视化
│
└── 文档
    ├── 快速开始.md               # 快速开始指南
    ├── README_v4改进说明.md      # v4改进说明（参考）
    └── README_v5改进说明.md      # v5改进说明（推荐阅读）
```

## 版本对比

| 特性 | v3 | v4 | v5 |
|------|----|----|-----|
| 目标函数 | 单一RMSE | 6项加权 | 4项简化SMAPE |
| 低基数处理 | ❌ 会爆炸 | ❌ 会爆炸 | ✅ SMAPE稳定 |
| 重复计分 | ✅ 无 | ❌ 有 | ✅ 已消除 |
| 权重可控性 | ✅ 简单 | ❌ 嵌套难控 | ✅ 直观 |
| 增长率限制 | 无 | 30%（太严） | 50%（合理） |
| 滚动回测 | ❌ 无 | ❌ 无 | ✅ 有 |
| 诊断工具 | ❌ 无 | ❌ 无 | ✅ 完整 |
| 推荐使用 | 快速测试 | ❌ 不推荐 | ✅ 生产使用 |

## 使用建议

### 场景1：快速测试
**使用v3**
```bash
python main_v3.py
```
- 优点：速度快，代码简单
- 缺点：精度有限，无诊断功能

### 场景2：需要高精度预测
**使用v5（必须先诊断）**
```bash
# 第1步：诊断
python diagnose_model.py

# 第2步：根据诊断结果决定是否继续
# 如果诊断显示"模型表现良好"，再创建v5主程序
```

### 场景3：对比不同版本效果
```bash
# 先运行v3和v4
python main_v3.py
python main_v4.py

# 再对比
python compare_v3_v4.py
```

## 文件清理建议

### 可以删除的文件
- `parameter_optimizer.py`（如果存在旧版本）
- `main_job_level.py`（如果存在旧版本）
- 任何临时测试文件

### 必须保留的文件
- 所有核心模型文件（job_model.py等）
- v3和v5的优化器和主程序
- 诊断工具（diagnose_model.py）
- 文档文件

## 下一步行动

1. **立即运行诊断**
   ```bash
   python diagnose_model.py
   ```

2. **查看诊断结果**
   - 检查控制台输出
   - 查看生成的诊断图表

3. **根据诊断决定**
   - ✅ 模型良好 → 创建v5主程序
   - ⚠️ 结构问题 → 需要改模型
   - ❌ 不如baseline → 用简单模型
