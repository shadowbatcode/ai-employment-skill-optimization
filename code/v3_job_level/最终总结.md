## 立即行动（3步）

### 第1步：运行诊断（必须）
```bash
cd v3_job_level
python diagnose_model.py
```

这会告诉你问题是：
- ✓ 优化层问题（可以用v5改进）
- ⚠️ 结构层问题（需要改模型）
- ❌ 模型不适合（换简单模型）

### 第2步：查看诊断结果

诊断工具会生成：
1. **控制台报告**
   - 滚动回测结果
   - 残差分析
   - 参数稳定性
   - Baseline对比

2. **诊断图表** (diagnosis_*.png)
   - 拟合效果图
   - 残差分析图
   - 参数稳定性图
   - Baseline对比图

### 第3步：根据诊断决定

**情况A: "✓ 模型表现良好"**
→ 可以创建v5主程序，使用v5优化器改进所有职位

**情况B: "⚠️ 系统性偏差/漂移"**
→ 需要分段建模或检查数据口径变化

**情况C: "⚠️ 参数不稳定"**
→ 模型过于复杂，需要简化或固定某些参数

**情况D: "❌ 不如baseline"**
→ 当前模型结构不适合，建议用简单时间序列模型

## 核心理念

> **"优化器只能在'你给的模型族'里找最优。如果模型族本身表达不了真实机制，再好的优化器也只是'让错误看起来更顺眼'。"**

因此：
- v5的核心价值不是"更好的拟合"
- 而是**诊断工具**——帮你判断问题根源
- 避免在错误的方向上浪费时间
