# 优化求解器改造说明

## 概述

已成功改造优化求解器以适应最新规则，扩展支持：
- **9维技能向量**（原5维扩展为9维）
- **多权重评估方案**（支持加权评估）
- **灵活的技能分类系统**（技术、领导力、可持续性）
- **向后兼容性**（保留原5维模型功能）

---

## 核心改进

### 1. 扩展拉马克模型 (ExtendedLarmarckOptimizer)

**旧模型**：
```
G(2026) = G(2025) + Λ·(E·x)
```
仅支持固定维度，无加权评估。

**新模型**：
```
G^weighted(2026) = G(2025) + Λ·(E·x)
其中包含加权评估系统:
  w_0: 技术技能权重
  w_1: 领导力和协作技能权重
  w_2: 可持续性和全球视野权重
  (w_0 + w_1 + w_2 = 1)

优化目标（加权版）:
min Σ(w_k * ||G_k(2026) - Target_k||²) for k in [0,1,2]
```

### 2. 技能维度映射系统

支持灵活的技能分类：
```python
skill_map = {
    0: ("编程能力", "technical"),
    1: ("AI工具使用", "technical"),
    2: ("系统架构", "technical"),
    3: ("沟通协作", "leadership"),
    4: ("领导力", "leadership"),
    5: ("团队管理", "leadership"),
    6: ("AI伦理安全", "sustainability"),
    7: ("环境意识", "sustainability"),
    8: ("全球视野", "sustainability"),
}
```

### 3. 多权重评估方案

支持三种预定义的权重方案：

#### 综合发展 (Comprehensive)
```
w_0 (技术)    = 0.4
w_1 (领导力)  = 0.3
w_2 (可持续)  = 0.3
```
平衡各类技能发展

#### 社会贡献 (Social)
```
w_0 (技术)    = 0.3
w_1 (领导力)  = 0.1
w_2 (可持续)  = 0.6
```
强调可持续性和全球视野

#### 技术优先 (Technical)
```
w_0 (技术)    = 0.6
w_1 (领导力)  = 0.2
w_2 (可持续)  = 0.2
```
重点发展技术技能

## API 变更

### 类初始化

**旧API**:
```python
optimizer = LarmarckOptimizer(data)
```

**新API**:
```python
optimizer = ExtendedLarmarckOptimizer(data, skill_dimension_map=skill_map)
# 或使用别名保持兼容性
optimizer = LarmarckOptimizer(data, skill_dimension_map=skill_map)
```

### 目标函数

**旧API**:
```python
result = optimizer.objective_function(x)
# 返回: MSE
```

**新API**:
```python
# 无权重（等同旧模型）
result = optimizer.objective_function(x)

# 带权重评估
weights = {'technical': 0.4, 'leadership': 0.3, 'sustainability': 0.3}
result = optimizer.objective_function(x, weights=weights)
```

### 求解方法

**旧API**:
```python
result = optimizer.solve_constrained(bounds=(0, 1), budget_constraint=1.0)
```

**新API**:
```python
# 无权重
result = optimizer.solve_constrained(bounds=(0, 1), budget_constraint=1.0)

# 带权重
weights = {'technical': 0.4, 'leadership': 0.3, 'sustainability': 0.3}
result = optimizer.solve_constrained(
    bounds=(0, 1),
    budget_constraint=1.0,
    skill_weights=weights
)
```

## 使用示例

### 运行标准5维模型
```bash
python 优化求解器.py
# 或
python 优化求解器.py original
```

### 运行扩展9维模型（支持多权重）
```bash
python 优化求解器.py extended
```

### 程序化调用

```python
from 优化求解器 import ExtendedLarmarckOptimizer, create_extended_data_pack
import numpy as np

# 获取9维数据包
data_pack = create_extended_data_pack()

# 获取某职业类别
category_data = data_pack["STEM_DataScience"]
skill_map = category_data.pop('skill_map')

# 创建优化器
optimizer = ExtendedLarmarckOptimizer(category_data, skill_dimension_map=skill_map)

# 定义权重方案
weights = {
    'technical': 0.4,
    'leadership': 0.3,
    'sustainability': 0.3
}

# 求解
result = optimizer.solve_constrained(
    bounds=(0, 1),
    budget_constraint=1.0,
    skill_weights=weights
)

# 分析结果
analysis = optimizer.analyze_results(result)
print(f"MSE: {analysis['total_loss']:.6f}")
print(f"RMSE: {analysis['rmse']:.6f}")
print(f"最优投入: {result['x_star']}")
print(f"预测效果: {result['G2026']}")
```

## 技术亮点

### 1. 灵活的权重系统
- 支持任意权重组合（w_0 + w_1 + w_2 = 1）
- 支持嵌套权重评估
- 易于扩展新的权重方案

### 2. 向后兼容设计
- 原5维模型完全保留
- 通过别名保持旧API：`LarmarckOptimizer = ExtendedLarmarckOptimizer`
- 可无缝切换模型维度

### 3. 自适应维度系统
- 自动检测数据维度（5维、9维或任意维）
- 自动生成默认的技能维度映射
- 支持自定义维度映射

### 4. 优化器选择
- 无约束求解：`lstsq_unconstrained`
- 约束求解：`SLSQP`（支持盒约束和预算约束）
- 加权求解：`SLSQP_weighted`（支持权重约束）
